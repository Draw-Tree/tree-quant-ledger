## R3 · 2026-07-18 — Conviction 重新校準（tree_quant/3）＋ 同業百分位估值閘

### 一、Conviction：由線性搬尺改為 logistic 重映射

**問題。** 舊公式 `conviction = (score + 3) / 5` 有一個致命的零點錯位：
一棵**完全沒有證據**的樹（每片葉都是 ⚪ 暫無定論），自動獲得 0.60 的信念
——「無證據」被當成六成勝算送入 Kelly 公式。結果是全部 71 棵樹擠在
[0.51, 0.98] 之間，沒有一棵低過 0.5——量尺只有上半截在用，看起來
「個個都同意」。

**修正。** 核心一步，是把「分數 → 機率」的**線性搬尺**，換成一條錨在
0.40 先驗的 **logistic（sigmoid）曲線**。整棵樹自葉而上這樣結算：

```
葉:      verdict → s ∈ {+2, +1, 0, −1, −2, −3}
並列枝:  s̄ = Σ(葉砝碼 × s) ÷ Σ葉砝碼    （葉按業務重要性加權求平均）
鏈式枝:  葉按弱度升序 × reverse Fibonacci [8,5,3,2,1] 加權（soft-min）
枝信念:  p = σ( logit(0.40) + 1.5 × s̄ )
根信念:  各枝信念在 log-odds 空間、以 Fibonacci 砝碼合成
```

兩個參數，各有一句人話解釋：

- **先驗 0.40**——一個未經驗證的投資命題，盲猜對的機率。零證據的樹
  落在這裡，不再高於擲毫。
- **每級 1.5 log-odds**——每上落一級 verdict，賠率變 4.5 倍。
  全部已驗證 ≈ 0.93（永不當確定），全部已證偽 ≈ 0.007。

**一句必須誠實的話：根層的「對數意見池」，代數上等於把加權平均分直接
映射。** 因為每一枝的信念都出自同一條 logistic（同一先驗、同一 λ），在
log-odds 空間對各枝加權平均，等價於先把各枝分數加權平均（即 h0_score）、
再過一次同一條 sigmoid——兩者只差捨入誤差。保留分層 pool 的寫法，是為
日後各枝可能帶有**異質先驗**時預留擴充；但就現制而言，它**不是**一部
「證據疊加器」，而是各枝**意見的加權共識**。這正是合成「對同一個根命題
的多個判斷」時應有的語義——枝與枝是意見，取共識，不是獨立證據相加。

**Sigmoid 帶來的性質，與它們真正的來源。**

- **頂部遞減**：第五個 ✅ 的邊際貢獻小於第一個（sigmoid 兩端變平）。
  這一條確實由曲線本身而來。
- **證偽在 odds 尺上伸得比確認遠**：全部已證偽 ≈ 0.007，全部已驗證
  只到 ≈ 0.93。但要老實講清楚——這個不對稱**並非 sigmoid 給的**：由
  0.40 先驗起行一步，一個正面級（→0.75）移動的機率，其實**多於**一個
  負面級（→0.13）。真正令證偽更決定性的是兩處刻意設計：**verdict 分數
  尺本身不對稱**（已證偽 = −3，已驗證僅 +2），以及**鏈式判決的硬 kill**。
  證偽因此在賠率尺上伸得更遠、也更難翻身——這才是本系統證偽主義哲學
  真正的數學形狀所在。

**鏈式分支同場修訂：由「最弱一環獨裁」改為 soft-min。** 舊制鏈式分支
取最弱一葉（min），但鏈的分類本身會錯——Samsung 曾被標為鏈，抽查一句
「他做不到，可以找台積電代工」即告破功。硬 min() 會處決被誤標的分支。
新制將葉按弱度排序、以 reverse Fibonacci [8,5,3,2,1] 加權：最弱一環佔
主導砝碼（四葉時約 44%）但不再一票定生死。**判決層的硬規則保留**：
鏈中任何一葉被證偽，分支判決仍為「已證偽」、證偽機器照常觸發——
判決層吵鬧，注碼層留有餘地。

**全 fleet 實測影響（71 棵樹）：**

| | 舊制 | 新制 |
|---|---|---|
| 平均 | 0.728 | 0.620 |
| 範圍 | [0.51, 0.98] | [0.28, 0.92] |
| 散度（sd） | 0.117 | 0.177 |

高證據樹站得住（GLW 0.98→0.92、V 0.97→0.91）；證據薄的現形
（Samsung 0.56→0.33、PDD 0.53→0.28）；鏈重的落中間帶（NVDA 0.53、
AMZN 0.42——被最弱環拖住但不畢命）。組合層面：ENPH、ONDS 的
edge 轉負，將由引擎剔除——「不是個個都同意」由口號變成倉位。

### 二、估值：同業百分位錨定閘（comps_bench，試行）

新增 `comps_bench.py`——以 Excel QUARTILE.INC 百分位對每層同業計算
倍數帶，並執行六項機器檢查：

- **VR1 tier-float**：情境倍數浮出所錨定 tier 的實測帶外
- **VR2 circular-anchor**：估值輸入由現價反推（循環錨定）
- **VR3 mcap sanity**：市值量級與輸入不符
- **VR4 bear authenticity**：熊市倍數無具名歷史低谷錨
- **VR5 forward-basis lint**：倍數口徑（NTM/FY+1）混用
- **W1 bull double-up**：牛市同時計入重估與盈利而未申報

首次試行即抓到作者自己的估值：MU 的基準情境 13.0x 浮出 forward tier1
帶 [5.5, 8.6] 之外——閘有效。現為 opt-in（樹須有 `peer_set` 區塊），
分批推廣至全 fleet 中。

### 三、一如既往

被取代的舊公式原文保留於 `VALUATION_STANDARD.md` §5.5；引擎版本號
`tree_quant/2 → tree_quant/3`；每棵樹的 computed 區塊已全數重刻並通過
71/71 檢查。conviction 的絕對水平仍未經樣本外校準——本次修訂改善的是
**相對排序的誠實度**，不是預測能力的證明；後者由模擬組合實驗回答。
