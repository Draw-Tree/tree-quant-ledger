## R4·R5 · 2026-07-20–21 — 設計全貌：一個投資論點，如何變成一個注碼

> 本節為 R4·R5 的重寫版。不再逐項羅列本週的修正，而是把整個系統的設計
> 由頭到尾講一次——每一層解決什麼問題、用什麼公式、公式從何而來、修正
> 之後如何運作。本週的三項修正（估值錨、權重定義、注碼混合）與一次引擎
> 改寫（信念合成，含上行折減）已融入相應段落。所有數字可用本庫公開數據
> 重算，指令見文末複製清單。

### 起因

本週一調倉，系統把模擬倉的 10.8% 配置在一隻股票（AMBA）上，是第二大
持倉的兩倍，而系統對這隻股票的信念只有 0.58（本系統以 0.40 代表「完全
沒有證據」）。追查發現三個部件各有一個獨立的缺陷，疊加成這個結果。
修好之後回頭看，問題不在個別參數，而在各層之間的語義沒有對齊：權重
不回答後果，信念不隨證據累積，注碼看不見下行。與其逐項記錄補丁，
不如把對齊後的系統完整講一次。

### 全流程一覽

系統的工作是把一個投資論點變成一個注碼。中間五步，每步有明確的輸入
與輸出，每週循環一次：

```
論點 → ① 樹（可證偽的結構）
     → ② 判定（每週逐葉對照新證據）
     → ③ 信念（證據合成為一個機率）
     → ④ 目標價（三情境估值）
     → ⑤ 注碼（信念 × 賠率 → Kelly）
     → 對帳與校準（每個參數對應一組檢驗數據）
```

### 第一步：論點寫成一棵可證偽的樹

一個投資論點通常是一句話——「這家公司會由硬體商轉型為平台」。一句話
無法檢驗，所以第一步是把它拆開：根節點（H0）是論點本身；其下三至五條
**分支**，每條是論點成立的一個**必要條件**——它倒下，論點跟著倒；每條
分支再拆成幾片**葉**，每片是一條可檢驗的子假說，寫明證據來源與**證偽
條件**（具體數字、具體期限），例如「若連續兩季設計中標少於 N 件，此葉
證偽」。整個結構遵循 Popper（1959）的可證偽性原則：不能被推翻的陳述
不進入樹。

每週，自動化管道逐葉搜尋新證據（搜尋聚合與模型判讀兩段交叉，來源須過
質量過濾），把每片葉標為六值判定之一，換算分數：

| 判定 | 分數 |
|---|---|
| ✅ 驗證 | +2 |
| 🟢 趨勢向好 | +1 |
| ⚪ 暫無定論 | 0 |
| 🟡 趨勢轉差 | −1 |
| 🟠 接近證偽 | −2 |
| 🔴 證偽 | −3 |

分數尺刻意不對稱：底是 −3，頂只有 +2——推翻比支持有力。這個選擇貫穿
全系統，第三步會再出現。葉分數在分支內加權平均，得分支分數 s̄ ∈ [−3, +2]。
若分支設有「必要葉」（necessity_leaves），該葉一旦證偽，整條分支直接
判證偽，不做平均——只有通過替代性測試（沒有它，是否存在其他路徑令
分支成立？答案為否）的葉才可標記為必要。每次判定轉變連同日期與證據
入庫，供文末的校準環節使用。

### 第二步：權重＝證偽的後果

**問題。** 分支之間輕重不同：「技術根基崩塌」與「毛利率改善慢半年」
不應同等對待。舊制按重要性**排名**，再按排名套固定權重（8/5/3/2/1，
費氏數列）。缺陷有二：一，排名不回答「這條分支被推翻時，結論應該跌到
哪裡」——而這正是權重存在的目的；二，兩條分支可能同樣致命，排名卻
強迫分出先後。

**想法。** 權重不應記錄「誰比誰重要」，應直接記錄**後果**：這條分支
被證偽，整體信念應跌到什麼水平。權重若是一句這樣的陳述，就可以被
事後核對——說了斷了會跌到 0.12，斷了之後就檢查是不是 0.12。

**解法。** 「後果作為權重」在可靠性工程有現成定義。一九六〇年代美國為
導彈與核電系統發展故障樹分析，把「系統失效」與各零件失效畫成一棵樹；
Birnbaum（1969）定義零件重要度＝系統失效機率對該零件失效機率的敏感度。
本系統的研究樹在結構上就是一棵故障樹（結論是根、分支是零件），定義
直接套用。量度單位取 Good（1950）的「證據重量」：以對數勝算（logit）
為單位——logit(p) = ln(p/(1−p))，機率 0.5 對應 0，愈確定為真愈大、愈
確定為假愈負，好處是證據可以加減。做法：建樹時每條分支回答「若被證偽，
整體信念應跌到哪裡」，五級作答，權重由答案換算：

| 等級 | 證偽後應跌至 | 權重 = 與先驗的 logit 差距 |
|---|---|---|
| 致命 | 0.05 以下 | 2.5 |
| 重創 | 約 0.12 | 1.6 |
| 明顯受損 | 約 0.22 | 0.9 |
| 輕微 | 約 0.31 | 0.4 |
| 邊緣 | 約 0.37 | 0.15 |

換算示範（重創級）：logit(0.12) = −1.99，logit(0.40) = −0.41，差距
1.58，取 1.6。舊費氏權重歸一化後與此表前三級接近（1 : 0.63 : 0.38 對
1 : 0.64 : 0.34）——原本的排名直覺方向沒錯，缺的是可核對的定義。並列
自此合法：兩條分支同為致命，是一句合法的陳述。尚未遷移的舊樹，引擎按
對照表換算（8→致命、5→重創、3→明顯受損、2→輕微、1→邊緣）——費氏數值
不是 logit 單位，不得直接相加。

**結果。** 首批 17 棵樹（前十五大持倉加兩隻貼近熊市目標者）已按此重評，
70 條分支中 46 條與機械換算不同。最一致的發現：舊排名把「不容易翻轉」
與「翻轉了也不要緊」混為一談——證據厚的分支被降權，只因它看來穩固；
而等級問的只是**如果**翻轉，後果多大。其餘 55 棵分批進行。

### 第三步：信念＝證據相加

**問題。** 權重成為可核對的陳述後，我們寫了檢查工具 flip-test：逐條
分支模擬「全部子假說證偽」，看整體信念是否跌到承諾水平。對 72 棵樹
運行，**144 個承諾達不到**。原因不在權重，在合成方式——舊引擎把各分支
信念取加權**平均**。以 V 為例：四條分支權重 5/3/2/1，證偽權重 3 的
分支，它只佔總權重 3/11，其餘 8/11 停在高位，平均後整體只跌到 0.62，
而承諾是 0.22。「平均」這個運算本身不容許小權重分支造成大跌。

**想法。** 兩條路。一：保留平均，重解每棵樹的權重遷就承諾——但多條
承諾互相爭權重，方程常無解，且解隨其他分支每週的狀態變動，權重會
漂移。二：把「平均」改為「相加」——由先驗出發，每條分支按狀態加減
一段對數勝算；證據從此累積（三條分支支持，信心高於一條支持），而每條
分支證偽時扣減的幅度固定，與其他分支無關，承諾自然成立。維護者選擇
後者：證據應該累積。

**解法。** 新公式：

```
每分支貢獻 cᵢ = wᵢ × (分支分數ᵢ ÷ 3)；若 cᵢ 為正，再乘 0.5（上行折減）
logit(整體) = logit(0.40) + Σᵢ cᵢ
信念 = sigmoid(logit(整體))，限制在 [0.005, 0.95]
```

上行折減的理由：每條分支寫的是論點成立的**必要條件**——它倒下，論點
跟著倒，所以證偽扣足全額；但它成立，只代表論點少了一種死法，不代表
論點已被證明——必要不等於充分，所以驗證只記一半。驗算三個情形：

1. 零證據：分數全為 0，logit = −0.41，信念 = 0.40，起點不變。
2. 單一「重創」分支證偽：貢獻 = 1.6 × (−3÷3) = −1.6（負向，不折減）；
   logit = −0.41 − 1.6 = −2.01；信念 = 0.119 ≈ 承諾的 0.12。五級逐一
   驗算，全部一致——折減不碰負向，承諾原封不動。
3. 累積：一條「明顯受損」分支全部驗證，貢獻 = 0.9 × (2÷3) × 0.5 =
   +0.3，信念 0.474；三條同級全部驗證，貢獻 +0.9，信念 0.621——三條
   支持仍然高於一條，平均制做不到；只是每一步都比未折減時走得慢。

至此，第一步的分數尺（−3 對 +2）與本步的折減（×0.5）疊加，證偽與驗證
的力度比為 **3:1**——「大膽假設，嚴格求證」寫入公式本身。

**結果。** 重跑 flip-test：144 個偏離全部消除。全艦隊 72 棵重算：平均
信念 0.619 → 0.496，標準差 0.176 → 0.121——重心下移、分佈收窄。上限的
行為隨之改變：舊制統一封在 0.93；新制取決於證據量——四分支舊樹全數
驗證約至 0.65，要更高就要更多、更重的獨立分支同時成立；另設 [0.005,
0.95] 上下限，永不宣稱確定。已知代價如實記錄：相加假設分支互相獨立，
若兩條分支由同一件事驅動（例如都依賴「AI 需求持續」），同一份證據會
計兩次，信念偏高——處理方式見「自我修正」：累積數據檢驗，無實證增益
則撤回。

### 第四步：目標價＝與歷史上同狀態的公司對照

**問題。** 信念之外，注碼還需要賠率——三個情境（牛市／基準／熊市）
各一個目標價。AMBA 事故的估值根源：熊市目標 $63.73 竟高於當時股價
$61.84，等於模型宣稱這隻股票沒有下行空間。錯在對照組：三個情境的倍數
一直對照**今日同業**——牛市對照今日的 AXON（12.89 倍），但今日的 AXON
已完成轉型多年，反映的是另一個階段；熊市帶則被兩隻正處災難期的名義
同業（MBLY、CRNC）拖低，製造假警報。

**想法。** 情境描述的不是今天，而是「另一個狀態」：牛市＝公司完成轉型
之後，熊市＝需求崩塌的谷底。對照對象不應是今日的同業，而應是**歷史上
真正處於那個狀態的公司**——在那個時刻，市場實際付過什麼價。

**解法。** 決策科學稱之為**參照組預測**（reference class forecasting）。
Kahneman 與 Tversky（1979）指出，人從個案細節向外推演時系統性地過度
樂觀，修正方法是先看歷史上同類個案的實際結果分佈；Flyvbjerg（2006）
把它應用到大型基建預算審核，其後被英國財政部納入指引。本系統的熊市
規則本來已要求「具名歷史低谷錨」——即熊市側一直在做參照組預測，這次
把同一要求推廣到牛市，兩端一致。基準情境維持今日同業對照——「現狀
延續」本來就以今日為尺。具體四步：

1. 為情境寫明「原型與階段」（牛市＝「硬體公司被市場承認為平台的
   時期」；熊市＝「需求斷崖的谷底」）。
2. 找一至兩個真實案例（公司、年份、當時倍數），來源可查；取該時期的
   持續中位數，不取最高一日。
3. 計算案例當時相對板塊中位的比率（修正利率環境差異）。
4. 今日目標倍數 = 該比率 × 今日板塊中位。

機器檢查兩條：VR7——案例、年份、倍數、來源、日期缺一即錯；VR8——未做
板塊相對化，給出警告。歷史倍數多為 TTM 口徑（歷史前向倍數無公開
數據），高增長期前向倍數約低於 TTM 15–30%，換算時註明。

**結果。** AMBA 熊市重算：案例取 AMBA 自己 2018 年的需求斷崖（GoPro
訂單消失），谷底 4.69 倍營收（Macrotrends 季度數據；當年 Motley Fool
文章獨立記載約「六倍營收」，兩源一致）；扣除大額淨現金後約 3.5 倍
企業價值對營收。套入：3.5 × $420M = $1,470M；加淨現金 $277.8M；除以
43.9M 股 = **$39.8**——一個真實出現過、低於現價的水平，取代原本高於
現價的 $63.73。牛市同一程序：AXON 2019–21 被承認為平台時期的持續中位
約 12.7 倍，換算今日環境約 10.8 倍前向，目標 $118.3（原 $141.07）。
為全艦隊遷移準備的原型案例庫已建成：14 個歷史情節（八個牛市承認期、
六個熊市谷底），每個附倍數、來源與日期；查無公開數據者如實標記為
不可用。

### 第五步：注碼＝信念先過市場，再入 Kelly

**背景：Kelly 公式需要什麼原料。** Kelly（1956，貝爾實驗室）證明：對
一注勝率與賠率已知的賭局，令長期資本增長最快的注碼比例是

```
f* = (p × b − q × a) ÷ (a × b)

p = 勝率，q = 1 − p = 敗率
b = 贏幅（贏時資本增加的比例）
a = 輸幅（輸時資本減少的比例）
```

分子是期望回報——優勢愈大，注愈大；分母是賠率的乘積——注意當輸幅 a
趨近零（「幾乎不會輸」）時分母趨近零，注碼趨近無限大。這一點是後面
事故的伏筆。

**三個情境各自的角色。** 股票不是二元賭局，所以估值層先把未來離散成
三個情境，每個情境一個目標價、一個機率：

- **牛市目標**提供贏幅：b = (牛市價 − 現價) ÷ 現價。
- **熊市目標**提供輸幅：a = (現價 − 熊市價) ÷ 現價。
- **基準情境**是「現狀延續」之下的合理價，作用有三。一，承載大部分
  機率質量——世界多數時間不走極端，期望回報的主體由它貢獻；沒有基準
  情境，模型只剩兩個極端，期望回報會被高估。二，它是市場隱含機率的
  座標之一——現價在熊市、基準、牛市三個目標之間的位置，就是市場投給
  三個情境的票（下文詳述）。三，下單引擎本身是二元的（只接受 p、b、a
  三個數），基準情境經期望回報折入：取一個有效勝率 p_eff，使二元公式
  的期望回報恰好等於三情境的期望回報——p_eff = (ER + a) ÷ (a + b)。
  三個情境的資訊由此無損地傳入二元引擎。

**問題：AMBA 事故逐步重演。** 用事發當日的實際數字（現價 $61.84；
當時目標價 牛 $141.07／基 $95.92／熊 $63.73）：

1. **機率層缺陷。** 舊制把樹的總分映射成三情境機率，而映射方式令
   總分為正的樹，熊市機率一律等於零。AMBA 當日：牛 0.34／基 0.66／
   熊 0。全艦隊大多數樹總分為正——即注碼公式眼中，幾乎沒有一棵樹
   會跌。
2. **賠率層缺陷**（第四步講過的估值錯誤）。熊市目標 $63.73 高於現價
   $61.84，輸幅 a ≤ 0——估值層宣稱這注沒有下行。
3. **兩個缺陷送入公式。** q 側無懲罰（熊市機率零），分母又趨零（輸幅
   零），f* 發散——引擎內部原始值 5.68，經單名上限與歸一化後仍配得
   全倉最大的 10.8%。

一句總結：公式沒有錯，兩個輸入異口同聲說「這注不會輸」，Kelly 理性地
回答「那就全押」。

**想法。** 賠率層由第四步的歷史谷底錨修正。機率層需要一個「證據不足
時自動轉為保守」的機制。市場價格本身就是現成的保守基準——股價隱含
市場對三個情境的定價。理想的行為：證據多，用自己的判斷；證據少，
靠向市場；完全沒有證據，就等於市場，沒有優勢，不下注。

**解法：兩條現成公式，分別寫明，再融合。**

公式一，Bühlmann（1967）信度公式。保險精算處理「新保戶數據太少」的
標準方法：

```
估計值 = Z × 個體數據 + (1 − Z) × 群體先驗
Z = m ÷ (m + K)        m = 數據量，K = 常數
```

行為：m = 0 時 Z = 0，全用群體先驗（新保戶交平均保費）；數據累積，
Z 趨向 1，逐步過渡到用自己的紀錄。K 控制過渡速度。

公式二，Black 與 Litterman（1992，高盛）。當年的實際問題：把自家的
回報預測直接餵入組合優化器，輸出的組合總是走向極端倉位——優化器把
預測的每一分自信都用盡。修正：以市場均衡回報作先驗，私人觀點按信心
加權併入——

```
最終預期 = 信心加權混合（市場先驗，私人觀點）
```

（原式為矩陣形式，此處取其結構。）信心愈低，結果愈貼近市場，倉位愈
溫和。

融合點：Black–Litterman 指明了**結構**（市場先驗＋私人觀點＋信心
權重），但信心從何而來由使用者自定；Bühlmann 恰好提供了**信心的
公式**（Z 由數據量決定）。本系統兩樣原料都已在庫：私人觀點＝樹的三
情境機率；市場先驗＝現價反推的三情境機率；數據量 m＝樹的證據覆蓋
（已有判定的子假說權重佔比）。合成：

```
Z = m ÷ (m + 0.5)
p_used(每個情境) = Z × p_樹 + (1 − Z) × p_市場
```

p_市場的計法：現價在三個目標價之間的位置，線性分票。現價恰等於基準
目標，全票投基準；現價落在基準與熊市目標之間，票按距離比例分給基準
與熊市；貼近哪個目標，哪個情境得票多。AMBA 事發當日現價已**低於**
舊熊市目標——市場隱含＝熊市 1.0。市場全票投熊，樹卻說熊市機率零，
兩者背馳本身就是警號，因此另設一條預警：現價 ≤ 熊市目標 × 1.1，
持倉標記「須重新覆核估值」。

**結果：AMBA 由頭到尾逐步重算**（2026-07-21 庫內實數）：

1. 證據覆蓋 m = 0.843（84% 的子假說權重已有判定）→ Z = 0.843 ÷
   (0.843 + 0.5) = 0.628。
2. 樹的機率：牛 0.34／基 0.66／熊 0。
3. 修正後目標價（第四步歷史錨）：熊 $39.8／基 $95.92／牛 $118.3；
   現價 $61.84 在熊與基之間，偏近中間——市場隱含：牛 0／基 0.39／
   熊 0.61。
4. 逐情境混合：牛 = 0.628×0.34 + 0.372×0 = 0.21；基 = 0.628×0.66 +
   0.372×0.39 = 0.56；熊 = 0.628×0 + 0.372×0.61 = 0.23。
5. 期望回報 = 0.21×(+91%) + 0.56×(+55%) + 0.23×(−36%) ≈ **+42%**
   （舊制 +80%）。熊市機率由 0 變 0.23，輸幅由零變 36%——注碼公式
   第一次同時看見下行的機率與幅度。
6. 送入 Kelly：half-Kelly 試算注碼 **0.9%**（舊制：分母趨零，封頂前

### 自我修正：每個參數對應一組檢驗數據

三個方法的來源行業有一個共同做法：初始參數來自專家判斷，之後用實際
數據修正——核電業的零件失效率也是先用手冊數字，再按電廠運行紀錄逐年
修正。本系統照同一方式設三層檢驗：

1. **建樹時檢查（已上線）。** flip-test 在建樹或改權重時運行，核對每條
   分支的等級與引擎實際行為是否一致。第一次運行找出 144 個不一致，促成
   第三步的引擎改寫；改寫後為零。
2. **事件對帳（數據已開始累積）。** 校準表把每一次判定轉變與該分支的
   等級、及其後的實際股價對接——首個訊號見下文。
3. **統計重校（等待樣本）。** 事件足夠後，檢驗「致命」級證偽之後的股價
   表現是否確實差於「邊緣」級；若五級之間沒有可辨別的差異，等級常數
   按數據修正。

**貝葉斯視角：等級是先驗，每週證據把它變成後驗。** 第二步的衝擊等級
由人判斷寫下——貝葉斯統計稱之為**先驗**（prior）：動工前的最佳估計。
每週的判定轉變連同其後的實際股價，構成**似然**（likelihood）：世界對
這個估計的回應。兩者結合成**後驗**（posterior）：獲數據支持的等級
保留，被數據反駁的向實證數字收縮——收縮的幅度用第五步同一條信度公式
（同類分支的案例愈多，實證數字的決定權愈大）。換言之，Birnbaum 重要度
不是一次寫死的常數，而是一個每週被證據修正的估計——權重層本身在
自我校準，這正是把等級定義成「可核對的陳述」的回報。

**首個訊號（2026-07-21 校準表，964 個事件，54 隻股票）。** 判定下調

## R3 · 2026-07-18 — Conviction 重新校準（tree_quant/3）＋ 同業百分位估值閘

### 一、Conviction：由線性搬尺改為 logistic 重映射

**問題。** 舊公式 `conviction = (score + 3) / 5` 有一個致命的零點錯位：
一棵**完全沒有證據**的樹（每片葉都是 ⚪ 暫無定論），自動獲得 0.60 的信念
——「無證據」被當成六成勝算送入 Kelly 公式。結果是全部 71 棵樹擠在
[0.51, 0.98] 之間，沒有一棵低過 0.5——量尺只有上半截在用，看起來
「個個都同意」。

**修正。** 核心一步，是把「分數 → 機率」的**線性搬尺**，換成一條錨在
0.40 先驗的 **logistic（sigmoid）曲線**。整棵樹自葉而上這樣結算：

```
葉:      verdict → s ∈ {+2, +1, 0, −1, −2, −3}
並列枝:  s̄ = Σ(葉砝碼 × s) ÷ Σ葉砝碼    （葉按業務重要性加權求平均）
鏈式枝:  葉按弱度升序 × reverse Fibonacci [8,5,3,2,1] 加權（soft-min）
枝信念:  p = σ( logit(0.40) + 1.5 × s̄ )
根信念:  各枝信念在 log-odds 空間、以 Fibonacci 砝碼合成
```

兩個參數，各有一句人話解釋：

- **先驗 0.40**——一個未經驗證的投資命題，盲猜對的機率。零證據的樹
  落在這裡，不再高於擲毫。
- **每級 1.5 log-odds**——每上落一級 verdict，賠率變 4.5 倍。
  全部已驗證 ≈ 0.93（永不當確定），全部已證偽 ≈ 0.007。

**一句必須誠實的話：根層的「對數意見池」，代數上等於把加權平均分直接
映射。** 因為每一枝的信念都出自同一條 logistic（同一先驗、同一 λ），在
log-odds 空間對各枝加權平均，等價於先把各枝分數加權平均（即 h0_score）、
再過一次同一條 sigmoid——兩者只差捨入誤差。保留分層 pool 的寫法，是為
日後各枝可能帶有**異質先驗**時預留擴充；但就現制而言，它**不是**一部
「證據疊加器」，而是各枝**意見的加權共識**。這正是合成「對同一個根命題
的多個判斷」時應有的語義——枝與枝是意見，取共識，不是獨立證據相加。

**Sigmoid 帶來的性質，與它們真正的來源。**

- **頂部遞減**：第五個 ✅ 的邊際貢獻小於第一個（sigmoid 兩端變平）。
  這一條確實由曲線本身而來。
- **證偽在 odds 尺上伸得比確認遠**：全部已證偽 ≈ 0.007，全部已驗證
  只到 ≈ 0.93。但要老實講清楚——這個不對稱**並非 sigmoid 給的**：由
  0.40 先驗起行一步，一個正面級（→0.75）移動的機率，其實**多於**一個
  負面級（→0.13）。真正令證偽更決定性的是兩處刻意設計：**verdict 分數
  尺本身不對稱**（已證偽 = −3，已驗證僅 +2），以及**鏈式判決的硬 kill**。
  證偽因此在賠率尺上伸得更遠、也更難翻身——這才是本系統證偽主義哲學
  真正的數學形狀所在。

**鏈式分支同場修訂：由「最弱一環獨裁」改為 soft-min。** 舊制鏈式分支
取最弱一葉（min），但鏈的分類本身會錯——Samsung 曾被標為鏈，抽查一句
「他做不到，可以找台積電代工」即告破功。硬 min() 會處決被誤標的分支。
新制將葉按弱度排序、以 reverse Fibonacci [8,5,3,2,1] 加權：最弱一環佔
主導砝碼（四葉時約 44%）但不再一票定生死。**判決層的硬規則保留**：
鏈中任何一葉被證偽，分支判決仍為「已證偽」、證偽機器照常觸發——
判決層吵鬧，注碼層留有餘地。
