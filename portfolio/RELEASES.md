### 引子：機器把 10.8% 押在一隻它不太確定的股票上

本週一調倉，系統把模擬倉 10.8% 押在單一隻股票上——第二大持倉的兩倍。
翻開帳目，它對這隻股票的信念只有 0.58（0.40 等於「完全無證據」）。它不是
特別有信心，只是賠率看起來太美：估值模型說下跌空間幾個百分點、上漲空間
超過一倍，注碼公式便照單全收。

當日我們按同業數據重錨了那隻股票。但事後檢討愈挖愈深：這不是一隻股票的
意外，而是三個部件各自帶著結構病——估值的尺借錯了對象、支柱的重量靠排名
而非後果、注碼公式會用賠率補償無知。

更意外的是三個解法的來歷。沒有一個出自金融教科書的估值章節：一個來自
核電廠的安全工程，一個來自二戰的密碼破譯室，一個來自保險公司的精算部門。
大概不是巧合——這三個行業，都以「錯一次就致命」為前提設計制度。

### 一、估值的尺：別問「同業今天值多少」，問「歷史上走到這一步的公司值多少」

**問題。** 牛市情境問的是：「如果這家公司蛻變成另一種東西，值多少？」
現制拿今天的成功者做尺——但今天的 AXON 是蛻變完成多年的成品，它的倍數
量的是別人的今天，不是這家公司的轉變時刻。名單裡還混入了正處於災難期的
「同業」，整條尺被拖低，製造假警報——本週事故的一半根源。

**想法。** 要量度轉變，就用轉變時刻的尺：牛市對照「歷史上真的完成過
這種蛻變的公司，在被市場承認的那段時期，實際拿到的定價」；熊市對照
真實崩盤的谷底。

**公式的出身。** 這個做法在決策科學裡有正式名字——**參照組預測**
（reference class forecasting）。Kahneman 與 Tversky（1979）指出：人做
預測慣用「內部視角」，從手上專案的細節向外推演，而內部視角系統性地
過度樂觀；解藥是先退一步問「歷史上同類事件的實際結果分佈長什麼樣」。
Flyvbjerg 後來把它工程化，用真實完工資料修正大型基建的預算，被英國
財政部納入評審指引——高鐵預算要先過「同類專案歷史上超支多少」這一關，
才輪到工程師自己的推演。

**為何剛好合身。** 我們的三個情境本來就是三個參照組問題：熊市＝
「這類故事失敗時，市場最後給過什麼價」；牛市＝「這類蛻變成功時，市場
實際給過什麼價」。而系統的熊市規則早已強制要求「具名歷史低谷錨」——
換言之，熊市側我們一直在做參照組預測，只是牛市側還借著今天的尺。本次
只是把同一紀律補到牛市，使兩端對稱；再加一道「制度歸一」，用相對倍數
消除利率時代不同的失真。

**用完之後。** AMBA 實測：舊熊市目標 $63.73 竟高於現價 $61.84——模型
眼中這隻股票「沒有下行」，正是注碼失控的燃料。改用它自己 2018 年崩盤
谷底作參照（4.69 倍營收；當年媒體獨立記載「six times sales」互證），
得出 **$39.8**——一個真實存在過、低於現價的谷底。牛市側由今日 AXON 的
12.89 倍，改為 AXON 2019–21 被承認期的持續中位（約 12.7 倍 TTM，相當於
當時軟體股中位的一倍上下），制度歸一後約 10.8 倍前向，目標 **$118.3**。

**數學。** 目標價公式不變（標準 §2.5）：

```
目標價 = (目標倍數 × 情境前向指標 + 淨現金) ÷ 稀釋股數      （ev 基準）
相對倍數ₜₕₑₙ = 類比事件倍數ₜₕₑₙ ÷ 當時 sector 中位倍數ₜₕₑₙ
目標倍數ₙₒw  = 相對倍數ₜₕₑₙ × 今日 sector 中位倍數
```

TTM→forward 換算：高增長期 forward 約低於 TTM 15–30%；收入下滑的谷底期
則略高。只取**持續中位**（≥6–12 個月），禁用峰值。

**實作步驟（可複製）。** ①為牛／熊寫明「原型＋階段」；②各找 ≥1–2 個
具名歷史事件，從公開歷史倍數表（如 Macrotrends 季度 P/S）取持續中位，
兩源互證；③記當時 sector 中位、算相對倍數（找不到就標 `UNAVAILABLE`
用絕對值並接受閘門警告）；④乘今日 sector 中位、做 TTM→forward 調整；
⑤熊錨必須是低於現價可及的真谷底；每項附 `source` ＋ `date`。

### 二、支柱的重量：一條為核反應堆寫的公式

**問題。** 每棵研究樹有幾條支柱。現制先按重要性排名，再套一組費氏數列
權重（8/5/3/2/1）。但排名回答不了真正的問題：**這條支柱斷了，整個論點
塌多少？** 兩條支柱可能同樣致命，排名卻硬要分先後。

**想法。** 權重不該是「誰比誰重要」，該是「斷了會塌多少」。

**公式的出身。** 一九六〇年代，美國要為導彈與核電系統回答同一條問題：
幾千個零件，哪一個失效最致命？工程師發明了**故障樹分析**——把「系統
失效」畫成樹根，零件失效畫成枝葉，逐層推演。統計學家 Birnbaum（1969）
為它補上量度：**一個元件的重要度 = 系統失效機率對該元件失效機率的
敏感度**。半個世紀後，核電安全評估仍在用這條公式。

量度「塌多少」的單位，另一段歷史早準備好了。二戰布萊切利園破譯
Enigma 時，圖靈需要量化「一條線索把假說推前多遠」，戰後他的統計助手
I. J. Good（1950）把方法整理成**證據重量**（weight of evidence）——
以對數勝算之差為單位。

**為何剛好合身。** 把我們的研究樹倒轉看：H0（論點失效）是樹根，分支
是元件——它在結構上**就是一棵故障樹**，Birnbaum 重要度翻譯過來正是
「此支柱反轉，H0 移動多少」。而我們的信念引擎本來就在對數勝算空間運算
（R3），Good 的單位是它的原生單位——兩條公式一條管結構、一條管單位，
恰好接上，不需改動引擎數學。

還有一段插曲。把沿用至今的費氏權重歸一化，與新的五級衝擊表並排：
前三級幾乎重合（1 : 0.63 : 0.38 對 1 : 0.64 : 0.34）。原來直覺一直在
暗中逼近一條真公式，只是沒有名字、沒有單位、無法檢驗。現在每個權重
都成了可事後對帳的承諾——你說斷了會跌到 0.12，斷了之後就查是不是 0.12。

**用完之後。** AMBA 權重 [2, 5, 3, 1] 依映射換成 [0.4, 1.6, 0.9, 0.15]，
信念 0.5798 → **0.5521**：證據最厚的支柱佔比升、邊緣支柱近乎歸零——
分佈更誠實，量級變化溫和。同時「鏈式／並列」兩套規則可以退役：所謂鏈，
就是每一環都致命，一套衝擊等級已能表達（無替代路徑的環另標
`necessity`，判決層的硬性熔斷保留）。

**數學。** 以 logit(p) = ln(p/(1−p)) 為尺：

```
wᵢ = | logit(H0 若支柱 i 證偽後應處水平) − logit(先驗 0.40) |
H0 信念 = σ( Σᵢ wᵢ·logit(cᵢ) ÷ Σᵢ wᵢ )     （σ 為 sigmoid；cᵢ 為分支信念，見 R3）
```

| 等級 | 證偽後 H0 應跌至 | wᵢ = \|Δlogit\| |
|---|---|---|
| 致命 | ≤0.05 | ≈2.5 |
| 重創 | ~0.12 | ≈1.6 |
| 明顯受損 | ~0.22 | ≈0.9 |
| 輕微 | ~0.31 | ≈0.4 |
| 邊緣 | ~0.37 | ≈0.15 |

**實作步驟（可複製）。** ①每條分支答「若被證偽，H0 應跌到哪」，對表取
wᵢ；②葉對分支同一套表；③權重寫入 `branches.yaml` 附 `impact_grade` 與
理由；④遷移映射：8→致命、5→重創、3→明顯、2→輕微、1→邊緣。

### 三、注碼的煞車：保險公司一百年前就解過這道題

**問題。** 注碼用 Kelly 公式。它同樣有出身：一九五六年貝爾實驗室的
Kelly 把資訊理論翻譯成「賭注應佔本金的比例」，Thorp 後來帶著它先贏
二十一點、再贏市場。但 Kelly 有一個沉默前提：**你輸入的機率是真的。**
當系統對論點沒有信心、而賠率碰巧極端（下檔小、上檔大），公式便用賠率
的美補償認知的空白。本週試點更證實這是全系統性的：現制機率映射下，
任何總分為正的樹，熊市機率恆為零——注碼公式從來沒有看見過下行。

**想法。** 信心不足時注碼要自動縮小——但「縮多少」需要一條有紀律的
公式，不是憑感覺打折。

**公式的出身。** 第一條來自保險業。精算師的百年難題：新保戶數據太少，
直接用他的理賠紀錄定價會被小樣本欺騙，完全不用又浪費資訊。瑞士精算師
Bühlmann（1967）給出答案：**按數據量加權，混合個體經驗與群體經驗**——
數據愈少，愈靠向群體；這條「信度公式」至今是保費定價的骨幹。第二條
來自高盛。Black 與 Litterman（1992）要治資產配置模型的老毛病——對輸入
極度敏感、輸出常常荒謬——方法是**以市場均衡為先驗，私人觀點按信心
加權併入**：沒有觀點，就回到市場。

**為何剛好合身。** 兩條公式需要的原料，系統裡現成都有。「群體／市場
先驗」＝股價本身隱含的三情境定價——系統一直在計算它，只是從未用於
注碼。「數據量」＝每棵樹有多少片葉已有判定、多少仍是空白。把 Bühlmann
的信度接上 Black–Litterman 的結構：**證據愈薄，愈聽市場；證據愈厚，
才愈敢與市場對賭；零證據時，注碼自動歸零**——沒有與市場不同的看法，
就沒有優勢，就不該有注。這與本實驗「敢買人人看死的股票」並不矛盾：
逆向的權利要由證據賺回來，不是由賠率白拿。

**用完之後（試點首日，平行計算、未影響下單）。**

| 股票 | 證據信度 Z | 注碼（舊制代理） | 注碼（新制代理） |
|---|---|---|---|
| COHR | 0.62 | 理論無界（下行盲區） | **0.85** |
| V | 0.67 | 理論無界（下行盲區） | **1.86**（證據最厚，注最大） |
| AMBA | 0.61 | 理論無界 | 單靠混合仍失控——因熊市目標高於現價，估值層根本沒有下行；補上問題一的歷史谷底錨後，三合一降至 **0.88** |

AMBA 一行正好證明三項變更**互相咬合**：混合機率治好機率盲區（COHR、V），
歷史谷底錨治好估值層的下行真空——單獨任何一項都不完整。

**數學。** 四步：

```
① 證據覆蓋  m = Σ_葉 (分支權重×葉權重)·1[判定 ≠ ⚪暫無定論] ÷ Σ_葉 (分支權重×葉權重)
② 信度      Z = m ÷ (m + k)                    （Bühlmann；試點 k = 0.5）
③ 入注機率  p_used = Z·p_verdict + (1−Z)·p_price_implied
④ 注碼      f* = argmax_f E[ln(1 + f·r)]，r ∈ {r牛, r基, r熊}；實際注 = ½·f*
```

`p_verdict` 為樹的證據機率（R3）；`p_price_implied` 為現價反推的市場
定價機率（Max-Base 錨定：現價落於熊–基之間時 p基 = (價−熊)÷(基−熊)、
p熊 = 1−p基、p牛 = 0；落於基–牛之間對稱處理）。下行 floor：r熊 ≤ −5%
（協議 §3.2）。極限性質：m→0 ⇒ Z→0 ⇒ p_used→市場定價 ⇒ 優勢→0 ⇒ f*→0。

**實作步驟（可複製）。** ①由 `hypotheses/*.json` 計 m（`pilot_v3.py` 之
`evidence_coverage()`）；②Z = m/(m+0.5)，k 愈大愈保守，待樣本足夠後以
校準數據定 k；③兩套機率皆在各樹 `computed.probabilities`，按 ③ 混合；
④三結果 Kelly 無閉式解，數值格點求 f*，取 half-Kelly；⑤現價 ≤ 熊目標
×1.10 → 標「須重錨覆核」。
**實例（AMBA）**：m = 0.788 → Z = 0.612；p_used = {牛 0.15, 基 0.62,
熊 0.24}；期望回報 +72.8% 修正至 +39.1%；f* = 1.76，half-Kelly 0.88。

## R3 · 2026-07-18 — Conviction 重新校準（tree_quant/3）＋ 同業百分位估值閘

### 一、Conviction：由線性搬尺改為 logistic 重映射

**問題。** 舊公式 `conviction = (score + 3) / 5` 有一個致命的零點錯位：
一棵**完全沒有證據**的樹（每片葉都是 ⚪ 暫無定論），自動獲得 0.60 的信念
——「無證據」被當成六成勝算送入 Kelly 公式。結果是全部 71 棵樹擠在
[0.51, 0.98] 之間，沒有一棵低過 0.5——量尺只有上半截在用，看起來
「個個都同意」。

**修正。** 核心一步，是把「分數 → 機率」的**線性搬尺**，換成一條錨在
0.40 先驗的 **logistic（sigmoid）曲線**。整棵樹自葉而上這樣結算：

```
葉:      verdict → s ∈ {+2, +1, 0, −1, −2, −3}
並列枝:  s̄ = Σ(葉砝碼 × s) ÷ Σ葉砝碼    （葉按業務重要性加權求平均）
鏈式枝:  葉按弱度升序 × reverse Fibonacci [8,5,3,2,1] 加權（soft-min）
枝信念:  p = σ( logit(0.40) + 1.5 × s̄ )
根信念:  各枝信念在 log-odds 空間、以 Fibonacci 砝碼合成
```

兩個參數，各有一句人話解釋：

- **先驗 0.40**——一個未經驗證的投資命題，盲猜對的機率。零證據的樹
  落在這裡，不再高於擲毫。
- **每級 1.5 log-odds**——每上落一級 verdict，賠率變 4.5 倍。
  全部已驗證 ≈ 0.93（永不當確定），全部已證偽 ≈ 0.007。

**一句必須誠實的話：根層的「對數意見池」，代數上等於把加權平均分直接
映射。** 因為每一枝的信念都出自同一條 logistic（同一先驗、同一 λ），在
log-odds 空間對各枝加權平均，等價於先把各枝分數加權平均（即 h0_score）、
再過一次同一條 sigmoid——兩者只差捨入誤差。保留分層 pool 的寫法，是為
日後各枝可能帶有**異質先驗**時預留擴充；但就現制而言，它**不是**一部
「證據疊加器」，而是各枝**意見的加權共識**。這正是合成「對同一個根命題
的多個判斷」時應有的語義——枝與枝是意見，取共識，不是獨立證據相加。

**Sigmoid 帶來的性質，與它們真正的來源。**

- **頂部遞減**：第五個 ✅ 的邊際貢獻小於第一個（sigmoid 兩端變平）。
  這一條確實由曲線本身而來。
- **證偽在 odds 尺上伸得比確認遠**：全部已證偽 ≈ 0.007，全部已驗證
  只到 ≈ 0.93。但要老實講清楚——這個不對稱**並非 sigmoid 給的**：由
  0.40 先驗起行一步，一個正面級（→0.75）移動的機率，其實**多於**一個
  負面級（→0.13）。真正令證偽更決定性的是兩處刻意設計：**verdict 分數
  尺本身不對稱**（已證偽 = −3，已驗證僅 +2），以及**鏈式判決的硬 kill**。
  證偽因此在賠率尺上伸得更遠、也更難翻身——這才是本系統證偽主義哲學
  真正的數學形狀所在。

**鏈式分支同場修訂：由「最弱一環獨裁」改為 soft-min。** 舊制鏈式分支
取最弱一葉（min），但鏈的分類本身會錯——Samsung 曾被標為鏈，抽查一句
「他做不到，可以找台積電代工」即告破功。硬 min() 會處決被誤標的分支。
新制將葉按弱度排序、以 reverse Fibonacci [8,5,3,2,1] 加權：最弱一環佔
主導砝碼（四葉時約 44%）但不再一票定生死。**判決層的硬規則保留**：
鏈中任何一葉被證偽，分支判決仍為「已證偽」、證偽機器照常觸發——
判決層吵鬧，注碼層留有餘地。
