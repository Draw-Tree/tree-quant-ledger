## 欄位字典（events.csv）

- `ticker` / `hypothesis` / `branch` — 事件所在的樹、子假說 ID、所屬分支 ID
- `impact_grade` — 分支衝擊等級（§1.5 封閉枚舉）；`grade_implied_from_fib` 為 True 表示等級由舊制 Fibonacci 權重推得（遷移映射），非人工判定
- `branch_weight` — 分支 log-odds 合成權重（等級對照表見 README）
- `date` / `from` / `to` / `score_delta` / `direction` — 轉變日期、前後判定（六值枚舉）、分數差（✅+2／🟢+1／⚪0／🟡−1／🟠−2／🔴−3）與方向
- `row_date` / `conviction_at` / `price_at` — 事件對齊到 quant_history 的首個不早於事件日的行：該行日期、當時 H0 信念、當時快照價
- `fwd_1w` / `fwd_4w` — 由該行起 +1 行／+4 行的快照價變化（每週一行，約等於 1 週／4 週；歷史不足時為空）

## 注意事項（誠實聲明）

1. **樣本少、歷史短**：追蹤僅數週，4 週前向回報大多尚未成熟。
2. **事件重疊**：同一週同一股票多個事件共享同一段前向回報，並非獨立樣本；尚未做顯著性檢定與市場回報扣減。
3. **前向回報未對沖大市**：讀數為原始回報，非超額回報。
4. **等級來源混合**：部分事件的等級為遷移映射之隱含值（`grade_implied_from_fib=True`），非當時人工判定。
